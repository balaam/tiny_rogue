Note: these animation guide-lines are only suggestions. Feel free to use the assets as you see fit.

ROW 1 - Idle

 

-Idle frames in the WonderDot style always respect the 16x16 grid, so that all units are easily visible while stationary.

-Ping-pong style animation. 1 - 2 - 3 - 2 (and repeat)

-Frames 1 and 3 require a longer delay than frame 2.
You may prefer a longer or shorter pause at frame 1 & 3, depending on the style you’re after.

ROW 2 - Selected / Action

Use same configuration as idle animation.

ROW 3, 4, 5 - Movement (down, side, up)

 

-Simple loop animation. Frames have a delay of 100ms in this example. You may want to tweak this number according to his movement speed.

-Flip side frames for left facing animations.

Notes:

  • Slime attack frames already extend quite a bit, so I wouldn’t recommend offsetting them like I have in this example.
  • Depending on the type of game you’re making, I recommend adjusting these values.
    For an action game, the player’s actions should be responsive (take out the delay in frame 1), while the enemies should give the player time to respond (increase the delay in frame 1).

ROW 9,10,11 - Hit

Frame order: 1,2,3,1

-Frame 1 is build-up frame. Give it a little extra delay to build anticipation for the hit.

-Frame 2, 3 are the hit frames.

-Ease back with frame 1 again.

Frame 4 is an all-purpose idle frame. Use these as you like.

A: 1,2,3,1 standard hit animation. 120ms for initial frame, 80ms for the rest.

B: Looping 2,3 one more time. 1,2,3,2,3,1.

C: Adding a flash to emphasis the impact. 1,2,3,2,3,1. (flash in bold)

You can loop frames 2,3 indefinitely to extend the animation. Could be useful if being hit by a long spell animation (etc.).
